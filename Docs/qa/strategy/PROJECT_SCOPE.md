# PROJECT SCOPE

## 1. Objective

Define the testing scope for OWASP Juice Shop from a QA perspective, focusing on the main business functionalities that will be validated through manual and automated testing.

---

## 2. In Scope

The following functionalities are included in the scope of this project:

* User authentication (login and registration)
* Product browsing and search functionality
* Shopping cart operations
* Checkout process (if applicable based on authentication)
* Basic user session handling

---

## 3. Out of Scope

The following areas are not part of the current testing scope:

* Informational pages (About Us, Photo Wall, Getting Started)
* External links (GitHub, Feedback, Contact Customer)
* Non-critical UI content and static pages
* Exploratory testing outside defined requirements

---

## 4. Notes

The scope is focused on core business flows that represent real user interactions.

It may be updated as new requirements or test scenarios are defined.
